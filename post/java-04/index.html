接口

接口中是抽象的，要通过子类里的函数来实现具体功能

  1. 接口是Java中的一种特殊的数据类型
  2. 定义接口的关键字是interface
  3. 接口不可以被实例化，即不可以new一个新接口，
  4. 接口中不可以有成员变量、构造函数、成员函数
  5. 接口中只能有抽象函数（虚函数）
  6. 接口的作用用于定义两个组件之间的组合对接标准
  7. 接口用于数据类型（可以用接口作为成员变量的类型、方法参数的类型等）
  8. 接口可以定义子类实现接口，关键字是implement
  9. 子类实现接口后，必须实现接口中的所有抽象函数
  10. JDK8以后接口的新特性：接口中可以定义default默认函数和static静态函数

什么是抽象函数

抽象函数指定函数没有大括号函数体，即函数没有实现过程
具体实现指的就是对抽象函数给出实现过程

如果父类中的函数不能帮子类实现任何流程，去除这个函数就会造成向上后必须向下。
所以你必须要定义这个函数，而这个函数就是抽象函数，他不实现任何功能，却参与了功能的实现。

  • 抽象函数用关键字abstract修饰，后米娜不可以写大括号，在参数的小括号后面分号结束
  • 如果父类中出现了抽象函数，则父类必须用abstract来修饰抽象类
  • 父类的抽象函数子类必须重写，不然要你子类有何用？子类需要帮助父类来完成父类没有实现的功能。
  • 父类的成员函数子类可以不用重写，因为可以继承

子类继承父类与子类实现接口有什么区别

  1. 继承父类的关键字extends，实现接口的关键字用implements

  2. 一个子类只能继承一个父类，一个子类可以实现多个接口

    public class A extends B{}
    //继承一个父类是对的

    //同时结成两个父类是错的

    //实现一个接口需要A实现B里面的抽象方法

    //可以同时实现两个接口，此时A可以当作B，也可以当作C使用
    public class A implements B，C{}

多态

向上转型

  • 向上转型时java中内置的一种机制，多态的运行离不开向上转型的机制
  • 运行向上转型的前提是传递的对象与目标类型存在继承或者实现关系
  1. 子类对象可以转化为父类的类型
  2. 子类对象可以转化为接口的类型
    eg
  3. new Dog对象传递给Pet类型
  4. new WeiXinPay对象 传递给Pay类型

向上后不用向下转型的前提

condition

  1. 父类有这个函数
  2. 子类要对这个函数进行重写

父类没有，就成为了子类的特殊函数，
子类不重写，就成了子类直接继承父类

抽象类和抽象函数的关系

  • 抽象类可以不包含抽象函数
  • 但是类有抽象函数，那你肯定是抽象类

抽象类和接口的区别

  • 抽象类和接口都不可以被实例化（就是创建对象）
  • 抽象类是一个类

四大权限修饰符

权限修饰符 当前类 同包的类 子类 不同包的类
private 不可见 不可见 不可见 不可见
（default） 可见 不可见 不可见 不可见
protected 可见 可见 不可见 不可见
public 可见 可见 可见 可见

权限修饰符的注意事项

  1. 子类继承父类，子类只能继承父类中非私有的成员
  2. 抽象函数的权限不能用private，用了之后就不能重写
  3. 构造函数可以用private修饰，对外不可见，外部就不可以创建这个类的对象了
  4. 不是所有的类都可以用private修饰，内部类可以被私有
    可见 = 暴露
    不可见 = 隐藏
