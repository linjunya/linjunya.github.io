  1. JDK、JRE、JVM的关系 （要说出3者的名字、英文和中文、以及包含关系）
    JDK（Java Development Kit）JDK是Java开发工具包，是Java程序员进行Java开发的核心环境。它包含了Java编译器（javac）、Java虚拟机（JVM）、Java类库（APIs）以及其他开发工具，如调试器（jdb）和文档生成器（javadoc）等。
    **JRE（Java Runtime Environment）**JRE是Java运行时环境，它包含了Java虚拟机（JVM）和Java核心类库。
    **JVM（Java Virtual Machine）**JVM是Java虚拟机，是Java平台的核心。它是一个抽象的计算机，能够执行Java字节码，实现Java程序的跨平台特性

  1. Java跨平台的原理 （至少要说出字节码技术，虚拟机技术，一次编译到处运行的原理）

Java跨平台原理概述
Java作为一种广泛应用的编程语言，其最大的特点之一就是“一次编译，到处运行”(Write Once, Run Anywhere, WOLA)的能力。这种能力主要得益于Java字节码技术和Java虚拟机(JVM)的设计。Java源代码首先被编译成字节码，然后在任何安装了Java虚拟机的环境中都可以不加修改地运行，从而实现了跨平台的特性。

字节码技术
Java源代码在被编译成字节码(.class文件)后，这些字节码不依赖于特定的硬件平台，因为它们是中间形式的代码，介于源代码和机器代码之间。字节码文件由一系列指令组成，这些指令在不同的平台上由Java虚拟机解释执行。

Java虚拟机技术
Java虚拟机是一个软件，它提供了一种运行Java字节码的环境。JVM在不同的操作系统上提供了相同的接口，因此编译后的字节码可以在任何支持JVM的系统上运行。JVM负责将字节码转换为特定平台的机器指令，这样就屏蔽了不同硬件和操作系统之间的差异。

结论
Java的跨平台性是通过Java字节码技术和Java虚拟机技术共同实现的。字节码技术允许Java源代码编译成中间形式的代码，而JVM技术则负责将这些字节码转换为可在各种硬件和操作系统上执行的机器指令。这种设计使得Java程序可以在几乎所有支持JVM的平台上无缝运行，大大提高了Java语言的普及度和实用性

  8. 解释什么是原码、反码、补码，以及他们的换算过程
    原码 (Sign-Magnitude Representation)
    原码是最直接的表示方法，它简单地使用最高位表示符号位（0 表示正数，1 表示负数），其余位表示数值部分。例如：

正数 5 的原码：00000101
负数 -5 的原码：10000101
换算过程：

将一个十进制整数转换为原码，直接用二进制表示，符号位为 0 表示正数，1 表示负数。
反码 (One's Complement)
反码是将原码中正数保持不变，负数的各位取反（0 变为 1，1 变为 0）得到的结果。例如：

正数 5 的反码：00000101
负数 -5 的反码：11111010
换算过程：

正数的反码和原码相同。
负数的反码可以通过将原码的符号位保持不变，其余位取反得到。
补码 (Two's Complement)
补码是计算机中最常用的表示有符号整数的方式，它解决了原码和反码的一些问题，例如在加法运算中的简便性。补码的表示方法是将反码表示法中负数部分的最低有效位加 1。例如：

正数 5 的补码：00000101
负数 -5 的补码：11111011
换算过程：

正数的补码和原码相同。
负数的补码可以通过将反码中的各位取反，然后在末位加 1 而得到。
换算示例
以整数 -5 为例：

原码：10000101
反码：11111010
补码：11111011
从补码转回原码或反码，同样是按照相反的过程进行计算，即补码减 1 后，反码中各位取反。
10. 解释什么是自动转换，什么是强制转换

自动转换（Implicit Conversion）：

自动转换指的是在不需要显式指示的情况下，由编程语言自动完成的数据类型转换过程。这种转换是安全的，不会导致数据丢失或不一致。例如，将一个整数赋值给一个浮点数变量，或者将一个较小的整数类型赋值给一个较大的整数类型变量时，编程语言会自动执行类型转换以确保兼容性。
强制转换（Explicit Conversion）：

强制转换是程序员有意识地指示编译器或解释器执行的数据类型转换。这种转换通常需要使用特定的语法或函数来明确表明数据类型的转换意图。强制转换可能会导致数据丢失或截断，因此需要谨慎使用，确保转换的安全性和合理性。

规则

  1. byte -> short -> int -> long -> float -> double
  2. char -> int -> long -> float -> double

从左到右是自动转换，从右到左得强制转换
